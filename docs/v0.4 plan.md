# E2W Phase 1 Executable Plan (v0.4)

## Adapter / Control Bootstrap with Cycle-Compatible Interface Reservation

---

# 0. 总目标

把当前 E2W v0.3 的 Wan-VACE quadmask 路径从：

```text
96 → 416 widened monolithic Conv3D
```

修订为：

```text
frozen legacy VACE path
+
trainable E2W Gated Causal Control Branch
```

Phase 1 的目标不是最终视觉质量，也不是 in-the-wild cyclic training。

Phase 1 的目标是证明：

```text
1. quadmask 真正进入可训练控制分支；
2. add/remove operation 真正影响控制分支；
3. legacy VACE path 没有被训练污染；
4. 小规模 paired/self-insertion anchors 可以 overfit；
5. Q0/Q1/Q2/Q3 的控制信号可被模型区分；
6. Q3 keep region 能被保持；
7. 架构接口未来能接 add→remove / remove→add-back cyclic supervision。
```

# 3. 关键修订：416-channel 语义必须说清楚

当前 v0.3 中：

```text
legacy VACE context input dim: 96
E2W packed quad context input dim: 416
```

这里的 `416` 指的是：

```text
pre-adapter / pre-embedding packed VACE context input channels
```

也就是进入原 widened patch embedding 之前的 raw context input channel count。

重构后不要错误地要求 residual output 仍然是 416 channels。

正确要求是：

```text
1. v0.3 兼容的 packed context construction 仍能构造 416-channel raw input。

2. split adapter 后：
   - legacy path 接收 x[:, :96]
   - quad/control branch 接收 x[:, 96:416] 以及 operation embedding
   - quad/control branch 的输出 shape 必须匹配 legacy VACE patch-embedding output shape

3. residual sum 发生在 embedded context space，而不是 raw 416-channel input space。
```

目标结构：

```python
x_legacy = x[:, :96]
x_quad   = x[:, 96:416]

legacy_context = frozen_legacy_patch_embedding(x_legacy)

quad_context = trainable_e2w_control_branch(
    x_quad=x_quad,
    operation=operation,
    inverse_operation=inverse_operation,   # optional / null in Phase 1
    object_side_packet=object_side_packet, # optional / null in Phase 1
)

context = legacy_context + gate * quad_context
```

# 4. 架构执行方案

## 4.1 新建模块

建议新增文件：

```text
tools/e2w_vace_control_branch.py
```

或者放入现有 helper 旁边：

```text
tools/e2w_vace_quad_i2v.py
```

但建议单独建模块，便于测试和后续 cycle trainer 复用。

模块职责：

```text

2. 接收 E2W extra packed masks / quad channels。
3. 接收 operation embedding。

6. 输出与 legacy VACE patch embedding 输出 shape 一致的 control residual。
7. 通过 non-dead residual gate 注入 legacy context。
```

```python
        reserve_cycle_fields=True,

        ...
```

内部至少包含：

```text
1. frozen legacy_patch_embedding
2. trainable quad_adapter
3. trainable operation_embedding
4. optional inverse_operation_embedding
5. optional object_side_packet projection stub
6. trainable residual gate
```


## 4.4 legacy path freeze 必须硬验证

最佳实现：

```python
legacy_conv.weight.requires_grad = False
legacy_conv.bias.requires_grad = False
```

```text
4. 每一步 assert:
   max_abs_delta(old_96_slice) < tolerance
```

但推荐不要长期使用这个 fallback。

# 5. Quadmask 语义保持不变

## 5.1 quadmask 输入

```text
quadmask.npy shape: [T, H, W]
```

允许 raw values：

|value|meaning|
|--:|---|
| `0` |primary target region|
| `63` |primary and affected overlap|
| `127` |affected non-target region|
| `255` |keep / unchanged region|

必须在 uint8 casting 前验证 raw values。

错误示例：

```text
raw value = 256
raw value = -1
raw value = 128
raw value = 1
```

这些都必须 reject。

---

## 5.2 semantic masks

继续构造：

```python
editable = quadmask in {0, 63, 127}
Q0 = quadmask == 0
Q1 = quadmask == 63
Q2 = quadmask == 127
Q3 = quadmask == 255
```

每个 extra mask 仍使用现有 VACE mask-packing path。

不要把 Q0/Q1/Q2/Q3 collapse 成 binary generation mask。

---

# 6. Binary generation mask 语义保持不变

Wan-VACE 仍然接收 binary generation mask：

|value|meaning|
|--:|---|
| `0` |keep / known source pixels|
| `255` |generate pixels|

Phase 1 默认使用：

```text
generation-mask-mode: quadmask-editable
```

定义：

```python
generate where quadmask != 255
keep where quadmask == 255
```

也就是：

```python
G = 255 where Q in {0, 63, 127}
G = 0   where Q == 255
```

注意：

```text
binary generation mask 只表示 known/generate gate。
semantic edit meaning 必须来自 quadmask + operation + prompt。
```

# 7. Operation / inverse-operation 接口

## 7.1 Phase 1 必须使用 explicit operation embedding

支持：

```text
operation: remove | add
```

不要只靠 prompt 文本表达 add/remove。

文本 prompt 可以辅助语义，但结构化 operation signal 必须存在并可训练。

---

## 7.2 为 cycle training 预留 inverse_operation

Phase 1 不训练 cycle loss，但 manifest 和 model forward 接口应预留：

```text
inverse_operation: add | remove | null
```

Phase 1 中可以全部为：

```text
inverse_operation: null
```

但接口要存在。

未来使用方式：

```text
add cycle:
  operation = add
  inverse_operation = remove

remove cycle:
  operation = remove
  inverse_operation = add
```

---

## 7.3 object_side_packet 接口预留

Phase 1 可以不用真实 side packet，但 manifest 和 forward 接口应支持：

```text
object_side_packet: null | dict
```

未来 remove→add-back cycle 需要它，否则模型可能通过 hidden information leakage 满足 reconstruction。

预留字段：

```json
{
  "object_crop_tube": null,
  "object_mask_tube": null,
  "appearance_embedding": null,
  "trajectory": null,
  "depth_or_contact_hint": null,
  "source_region": null
}
```

Phase 1 中可以全部为 null。

# 8. 初始化规则

避免 dead control path。

不能同时让：

```text
quad adapter final output = exact zero
residual gate = exact zero
operation embedding = exact zero
```

导致梯度无法有效进入控制分支。

允许三种初始化之一。

## Option A

```text
quad weights zero-init
residual scale = 1.0
```

## Option B

```text
quad weights small random init
residual gate alpha ≈ 1e-3
```

## Option C

```text
internal adapter random init
final projection zero-init but trainable
residual gate nonzero
```

执行者必须在日志中记录所选方案：

```json
{
  "control_branch_init": "Option B",
  "quad_adapter_init": "small_random",
  "residual_gate_init": 0.001,
  "operation_embedding_init": "small_random_or_zero_with_gradient"
}
```

# 9. 测试策略修订

因为 zero-init 可能导致训练前 final context 不随 Q/op 改变，所以测试分两类。

---

## 9.1 Pre-training structural tests

这些测试在训练前必须通过。

```text
1. v0.2/v0.3 existing contract tests pass。

2. invalid raw quadmask values rejected before uint8 casting。

3. pre-adapter packed context construction remains v0.3-compatible:
   raw packed context input can still be 416 channels。

4. split adapter output shape matches legacy patch embedding output shape。

5. legacy 96-channel path requires_grad=False。

6. quad adapter parameters require_grad=True。

7. operation embedding parameters require_grad=True。

8. one backward pass gives nonzero gradient to quad adapter。

9. one backward pass gives nonzero gradient to operation embedding。

10. one optimizer step does not change legacy 96-channel weights。

11. shape mismatch between video and quadmask is rejected unless explicit alignment is requested。

12. explicit alignment records:
    - source shape
    - target shape
    - method
    - deterministic frame mapping
```

---

## 9.2 Post-step / post-overfit behavioral tests

这些测试可以在 one-step 或 overfit checkpoint 后执行。

```text
1. changing Q0/Q1/Q2/Q3 changes final context or model output。

2. changing operation remove/add changes final context or model output。

3. Q3 region remains closer to source than editable regions。

4. Q0/Q1 shows visible add/remove response on anchor samples。

5. Q2 changes when anchor metadata marks affected-region change expected。

6. legacy path remains unchanged after training.
```

# 11. Anchor data audit gate

每个 sample 进入训练前必须通过 audit。

## 11.1 必须检查

```text
1. quadmask raw values only in {0, 63, 127, 255}

2. frame count is 4n + 1

3. src, target, quadmask temporal length aligned

4. src, target, quadmask spatial size aligned

5. 如果不 aligned，必须显式使用 nearest alignment

6. Q0 or Q1 non-empty for actual add/remove samples

7. Q3 non-empty

8. Q3 source-target difference below threshold unless explicitly exempted

9. Q2 non-empty only when downstream affected-region change is expected

10. prompt passes target-contamination check

11. operation is remove or add

12. anchor_type is allowed
```

---

## 11.2 Q3 consistency gate

因为 Q3 是 keep region，如果 source 和 target 在 Q3 本身差异很大，会产生矛盾监督。

必须计算：

```text
Q3_delta = distance(src[Q3], target[Q3])
```

可以先用简单指标：

```text
pixel L1 / latent L2 / VAE feature distance
```

Phase 1 至少记录：

```json
{
  "q3_source_target_delta": 0.012,
  "q3_source_target_delta_ok": true
}
```

如果 Q3 delta 太大：

```text
sample 不进入 Phase 1 training
```

除非 metadata 明确标注：

```text
q3_exception_reason
```

# 12. Self-insertion anchor 必须支持双向

已知 self-insertion remove pair：

```python
b = clean real background video
o = object tube from another source
x = compose(b, o, mask/depth/shadow)

remove:
  src = x
  target = b
  operation = remove
```

Phase 1 还需要 add examples，所以 self-insertion 应同时生成：

```python
add:
  src = b
  target = x
  operation = add
  object_side_packet = optional object tube / null in Phase 1
```

如果 Phase 1 暂时没有 side packet，仍应保证 add anchors 来自：

```text
1. synthetic paired samples
2. self-insertion reverse direction
3. manually verified add pairs
```

不要只用 remove anchors，否则 operation embedding 很难证明 add/remove sensitivity。

# 13. Prompt gate

Prompt contamination 不能只是备注，必须成为 gate。

已知：

```text
30 teacher-grounded eval rows:
  14/30 strict target-free VACE prompt pass
  16/30 fail

smoke samples 0076, 0077, 0128 fail due to target-contaminated candidate VACE text
```

执行规则：

```text
1. removal prompt 不得把要移除的物体描述为仍然存在。

2. add prompt 可以描述新增目标，但不得泄漏 target-specific artifacts。

3. prompt_id、source、target-contamination pass/fail 必须写入 metadata。

4. failed prompt 不进入训练，除非 manual_fixed 后重新标记 pass。
```

# 14. Training loop

Phase 1 使用和 inference 一致的 VACE context format。

伪代码：

```python
for batch in loader:
    src = batch["src_video"]
    tgt = batch["target_video"]
    Q = batch["quadmask"]
    op = batch["operation"]

    inverse_op = batch.get("inverse_operation", None)
    object_side_packet = batch.get("object_side_packet", None)

    txt = batch["prompt"]

    assert set(unique_raw_values(Q)) <= {0, 63, 127, 255}
    assert frame_count_is_4n_plus_1(tgt)

    Q = align_quadmask_explicitly_if_needed(
        Q,
        target_shape=tgt.shape,
        method=batch["alignment"]["method"],
    )

    G = build_generation_mask(
        Q,
        mode=batch["generation_mask_mode"],
    )

    E  = (Q != 255)
    Q0 = (Q == 0)
    Q1 = (Q == 63)
    Q2 = (Q == 127)
    Q3 = (Q == 255)

    packed_context = build_vace_context_input(
        src_video=src,
        generation_mask=G,
        semantic_masks=[E, Q0, Q1, Q2, Q3],
    )

    legacy_input = packed_context[:, :96]
    quad_input   = packed_context[:, 96:416]

    legacy_context = frozen_legacy_vace_patch_embedding(legacy_input)

    quad_context = e2w_control_branch(
        quad_input=quad_input,
        operation=op,
        inverse_operation=inverse_op,
        object_side_packet=object_side_packet,
    )

    context = legacy_context + e2w_control_branch.gate() * quad_context

    z0 = vae.encode(tgt)

    t, noise = sample_diffusion_or_flow_timestep()

    zt = scheduler.add_noise_or_interpolate(z0, noise, t)

    target = scheduler.training_target(z0, noise, t)

    pred = wan_vace(
        noisy_latents=zt,
        timestep=t,
        text=txt,
        context=context,
        operation=op,
    )

    loss_map = mse(pred, target, reduction="none")

    loss_noise = region_weighted_loss(loss_map, Q)

    loss_q3_preserve = q3_preservation_loss(
        pred=pred,
        src=src,
        target=tgt,
        mask=Q3,
    )

    loss = loss_noise + lambda_q3 * loss_q3_preserve

    loss.backward()

    assert_quad_adapter_grad_nonzero()
    assert_operation_embedding_grad_nonzero()

    enforce_legacy_grad_zero_if_needed()

    optimizer.step()

    restore_or_assert_legacy_weights_unchanged()
```

# 15. Loss

## 15.1 Minimum Phase 1 loss

使用 region-normalized loss，不用 raw pixel sum。

起始权重：

```text
L_noise =
  0.5  * mean_all(loss)
+ 3.0  * mean_Q0(loss)
+ 6.0  * mean_Q1(loss)
+ 4.0  * mean_Q2(loss)
+ 0.25 * mean_Q3(loss)
```

解释：

```text
Q0: primary target add/remove region
Q1: primary + affected overlap, highest priority
Q2: affected non-target region
Q3: keep / preservation region
```

---

## 15.2 Q3 preservation metric / loss 必须加入

Phase 1 至少要记录：

```text
Q3 pixel/latent preservation metric
```

推荐最小 loss：

```text
L_total = L_noise + λ_q3 * L_preserve_Q3
```

其中：

```text
L_preserve_Q3 = distance(predicted_Q3, source_Q3)
```

可以先用：

```text
latent L2
pixel L1
VAE feature distance
```

不要让 Q3 raw area domination 掩盖 Q0/Q1/Q2，所以 Q3 preservation 应单独 region-normalized。

---

## 15.3 Optional metrics

Phase 1 可以记录但不强制训练：

```text
L_object_Q0_Q1:
  add presence / remove absence metric

L_physics_Q2:
  affected-region metric where synthetic tracks or masks exist
```

```text

  - prove Q/op sensitivity
```

通过条件：

```text

2. outputs change when quadmask changes
3. outputs change when operation changes
4. Q3 remains closer to source than editable regions
5. Q0/Q1 shows visible add/remove behavior
```

---

## Stage 1C — Bounded sanity expansion

```text
samples: up to 100
goal:
  - verify control branch is not only memorizing one tiny batch
  - include both add and remove anchors
```

通过条件：

```text
1. no legacy drift
2. add/remove both represented
3. Q3 preservation metric remains acceptable
4. Q0/Q1 edit behavior still visible
5. Q2 changes only where expected
```
