# E2W 项目 Handoff 文档（2026-06-08）

## 1. 项目一句话目标

训练 VLM Planner 通过结构化反事实推理
（target_ref + counterfactual_state）
驱动 tool-augmented grounding（DINO+SAM2）
和 VACE 视频编辑，
验证 VLM 能学会物理世界表征。

## 7. 重要架构决策和理由

- 为什么改成 tool-augmented grounding
  - 因为仅靠 Planner 几何字段难以稳定提供可控的语义上下文；`tool_augmented` 在 `v8` 中把 fill_type、非目标效果维度写入 schema，便于统一验证与后续消融。
  - 但要强调：新增表达能力不等于可直接过 full-pass，核心约束仍在执行器 gate。
- 为什么 counterfactual_state 要 7 字段 + fill_type 枚举
  - 现有失败集中在 `if_removed` 污染与几何语义不一致。
  - 7 字段拆分（几何、材料/光照、交互、时间一致性）有助于对 `serialize_vace_prompt` 做精细验证并避免目标残留语义。
- 为什么 if_removed 要 target-free
  - VACE prompt 直接决定文本条件编辑风格；历史上目标泄漏会触发 `vace_prompt_contract_failed`，使可执行链条被硬拦截。
  - 因此契约明确：`without/no longer/removed` 等表达一律禁止，且只允许描述非目标影响。
- Phase 1A vs Phase 1B 的区别和必要性
  - 1A：冻结 legacy path，建立可训练 control branch 的结构完整性，验证梯度与更新链路；
  - 1B：小规模 overfit（16–32）验证 overfit 可行性、operation 敏感性与 Q0/Q1/Q2/Q3 信号；
  - 必要性在于先证明“能学得动且不污染 legacy”，再扩展数据规模（1C）与循环训练。
